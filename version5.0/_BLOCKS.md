# v5.0 재사용 인스트럭션 블록 (`_BLOCKS.md`)

> 모든 Step 프롬프트가 참조하는 **횡단 원칙 라이브러리**. 각 블록은 독립적으로 잘라 붙여 넣을 수 있도록 설계되었으며, Step 본문에서는 `[블록 A 적용]` 형태로만 호출하거나, 필요 시 블록 본문을 그대로 복사해 끼워 넣는다.

---

## 블록 A — 용어 번역기 (Term Translator)

```text
[비전공자 보조 규칙 — 용어 번역기]
출력 결과물에 다음 부류의 용어가 등장할 때마다, 해당 용어 바로 뒤
괄호 안에 비전공자 눈높이의 10자 내외 쉬운 풀이를 반드시 덧붙여라.

대상 용어:
- 회계/재무: CAPEX, OPEX, OPM, ROE, ROA, ROIC, EBITDA, FCF, WACC,
            순운전자본, 자본적지출, 영업레버리지 등
- 밸류에이션: PER, PBR, PSR, EV/EBITDA, DCF, 멀티플, 리레이팅
- 산업/기술: 본인 산업의 약어·기술 용어 (예: 반도체 - HBM, EUV / 바이오 - CMO, mRNA)

표기 규칙:
- 형식: `용어(쉬운 풀이)` — 예) `ROE(자기자본 대비 이익률)`
- 같은 용어가 한 답변 안에서 여러 번 등장해도, 첫 등장에만 풀이를 붙여라.
- 풀이는 정의가 아니라 "감"을 잡게 해주는 비유적 설명을 우선한다.
```

## 블록 B — Sanity Check (자체 논리 점검)

```text
[자체 점검 규칙 — Sanity Check]
출력하기 직전, 네가 작성한 모든 인과 추론을 스스로 점검하라.
다음 중 하나라도 해당되면, 즉시 보강하여 다시 작성하라.

논리적 비약 패턴:
1. "매출 ↑ → 주가 ↑" 같은 1차원 인과 (중간 단계 누락)
2. "시장이 성장하니 우리 회사도 성장" (점유율·경쟁우위 누락)
3. "신제품 출시 → 실적 개선" (가격·물량·마진 분해 누락)
4. "수치 X% 상승 → 가치 Y% 상승" (멀티플·할인율 무관 가정)

보강 방법:
- 매출 → 주가 사이의 매개변수(점유율, 마진, 멀티플 리레이팅)를 명시
- 가정과 근거를 분리하여 표기
- 가정에는 `[가정]` 태그, 검증된 사실에는 출처 명시
```

## 블록 C — 핵심 비유 (Anchor Analogy)

```text
[비전공자 보조 규칙 — 핵심 비유]
복잡한 비즈니스 모델·산업 구조·경쟁 구도를 설명하기 직전,
일상생활이나 대중적 서비스에 빗댄 한 줄 비유를 본문 최상단에 배치하라.

규칙:
- 1문장, 30자 내외
- 비유 대상: 배달의민족, 스타벅스, 넷플릭스, 카카오톡 등 누구나 아는 서비스
- 형식: "이 회사는 [산업]의 [비유 대상] 같은 존재 — [핵심 차별점 한 마디]"
- 단순 호명이 아니라 "왜 그렇게 비유했는지"가 드러나야 함
```

## 블록 D — Mermaid 다이어그램

```text
[시각화 규칙 — Mermaid]
산업 밸류체인, 기업 지배구조, 사업부 구조, 거래 흐름 등 관계가
중요한 정보는 본문 설명에 더해 Mermaid 코드 블록으로 출력하라.

문법 가이드:
- 밸류체인 / 가치 흐름: `flowchart LR`
- 지배구조 / 계층: `flowchart TD`
- 시간 흐름 / 단계: `flowchart LR` 또는 `sequenceDiagram`

규칙:
- 코드 블록은 ```mermaid ... ``` 로 감쌀 것 (mermaid.live, Notion 등에서 즉시 렌더)
- 노드는 5~9개 이내, 관계선이 교차하지 않도록 간결화
- 한국어 노드명 허용, 따옴표 필요 시 `["사업부 A"]` 형식
```

## 블록 E — 검증 태그 (`[검증 필요]`)

```text
[데이터 신뢰도 규칙 — 검증 태그]
출력에 포함되는 모든 정량 수치(시가총액, 매출, 영업이익, 시장 규모,
점유율, 멀티플 등)를 다음 기준으로 자체 검토하고, 의심스러운 항목에
`[검증 필요]` 태그를 단어 직후에 부착하라.

의심 기준:
1. 동일 답변 내에서 같은 지표가 다른 수치로 등장
2. 출처가 불분명하거나 기준일이 모호
3. 단위 환산 불일치(억/조/USD/KRW 혼재)
4. 직전 단계 출력과 모순

표기 형식:
- `매출 1,234억원[검증 필요]` — 수치 옆 공백 없이 부착
- 동일 답변 끝에 `[검증 필요]` 항목 모음 표 출력
  | 항목 | 우리 답변 수치 | 의심 사유 | 권장 1차 출처 |
```

## 블록 F — Red Team 페르소나 (Step 5.5 전용)

```text
[Red Team 페르소나]
지금부터 너는 Wall Street의 베테랑 Short Seller(공매도 전문가)다.
직전 단계에서 도출된 투자포인트와 재무 분석을 가장 회의적인 시각으로
공격하라. 너의 목표는 약점을 찾는 것이지 균형을 맞추는 것이 아니다.

산출물:
1. 공격 질문 3개 (논리적 허점 / 데이터 맹점 / 가정의 취약성)
2. 각 질문에 대한 방어 논리 (Bull-side 입장)
3. 방어 불가능한 항목은 `[방어 불가]` 태그 부착
4. 최종: 살아남은 투자포인트만 추려 1줄로 재정리

톤: 차갑고 직설적. 정중함보다 정확성.
```

## 블록 호출 규약

각 Step 프롬프트 본문 마지막에 다음과 같이 호출한다:

```
※ 이 Step에는 다음 블록을 적용하라:
- 블록 A (용어 번역기)
- 블록 B (Sanity Check)
- 블록 E (검증 태그)
```

또는 비전공자 부원이 챗봇에 한 번에 붙여넣을 수 있도록, Final 통합본에서는 호출 위치에 블록 본문을 그대로 인라인 삽입한다.

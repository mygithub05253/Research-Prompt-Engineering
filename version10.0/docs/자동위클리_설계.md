# 자동 위클리 갱신 설계 (v10.1+ 구현 예정)

> **상태**: v10.0에서는 **설계만** 박제. cron + scheduled-tasks MCP + 발송 채널 인증 검증 후 v10.1+에서 구현.
> **CLAUDE.md §7.4 규정**: "v10에서 다룰 예정"이라 명시 — 본 문서가 v10에서 다루는 결과물.

---

## 1. 비전

매주 월요일 KST 05:00 — 학회 등록 종목별로 직전 주 위클리 결과 + 신규 정보를 자동 갱신해 부원에게 발송.

```
[월요일 05:00 KST]
    ↓
[scheduled-tasks MCP cron 트리거]
    ↓
[학회 종목 리스트 (data/watchlist.yaml)]
    ↓
[종목별 신규 정보 수집]
  - korea-stock-mcp: 신규 공시 (직전 7일)
  - tavily/exa: 산업 뉴스 (직전 7일, 출처 URL)
  - pykrx-mcp: 주가·거래량 (W/W 변동)
  - 한경 컨센서스: EPS·목표주가 변화
    ↓
[종목별 부록 D yaml 생성]
    ↓
[/gic-weekly Skill 자동 호출]
  - templates/위클리_4p.html 적용
  - slides-grab → PDF + PNG
    ↓
[학회 드라이브 자동 업로드]
  /GIC_분석/위클리/[YYYY-WW]/[기업명]/
    ↓
[발송 채널]
  - Slack #gic-weekly 채널: PNG 4장 + PDF 링크
  - 이메일: GIC 메일링 리스트
    ↓
[부원이 PDF 검토 → 필요시 PowerPoint에서 직접 편집]
```

## 5. 신규 정보 수집 로직 (의사코드)

```python
# scripts/auto_weekly_yaml.py (예정)

def collect_weekly_data(stock_code: str, prev_yaml: dict) -> dict:
    """직전 7일 신규 정보를 수집해 부록 D yaml 생성"""
    new_yaml = {
        "slide_1_weekly_cover": {
            "메인_타이틀": "",  # 신규 정보 헤드라인 자동 생성
            "투자의견_박스": {},
            "변경_사유_1줄": "",
        },
        # ...
    }

    # 1. 신규 공시 (DART)
    new_disclosures = korea_stock_mcp.get_disclosure_list(
        corp_code=corp_code_from(stock_code),
        days=7,
    )

    # 2. 주가 변동
    ohlcv = pykrx_mcp.get_stock_ohlcv(stock_code, days=7)
    new_yaml["slide_1_weekly_cover"]["투자의견_박스"]["주가_변동_W주W"] = (
        f"{((ohlcv[-1].close / ohlcv[0].close) - 1) * 100:+.1f}%"
    )

    # 3. 산업 뉴스
    news = tavily_search(f"{stock_code_to_name(stock_code)} 최근 뉴스 1주",
                         days=7, with_urls=True)
    new_yaml["slide_2_new_info"]["뉴스_산업동향"] = [
        {"날짜": n.date, "내용": n.summary[:80], "출처": n.url}
        for n in news[:5]
    ]

    # 4. 컨센서스 변화 (한경 컨센서스 또는 FnGuide 크롤링)
    # ...

    # 5. 거시 변수 (한국은행 ECOS)
    # ...

    return new_yaml
```

## 6. cron 스케줄 (scheduled-tasks MCP)

```yaml
# .claude/scheduled/weekly_update.yaml (예정)
name: gic_weekly_auto
schedule: "0 5 * * 1"   # 매주 월요일 05:00 KST (UTC 20:00 일요일)
timezone: "Asia/Seoul"
command: |
  python scripts/auto_weekly_yaml.py --watchlist data/watchlist.yaml
  for yaml_file in data/output/_weekly_temp/*.yaml; do
    /gic-weekly --yaml "$yaml_file" --auto-upload
  done
on_complete:
  - script: scripts/send_to_slack.py
  - script: scripts/send_to_mailing.py
```

## 7. 발송 채널 (검토 필요)

### 7.1 Slack
- 학회 워크스페이스 워크플로 봇 설정
- 채널: `#gic-weekly`
- 형식: 종목별 PNG 4장 + PDF 링크
- 인증: Slack Webhook (SLACK_WEBHOOK_URL 환경변수)

### 7.2 이메일
- 학회 mailing list (예: `gic@gachon.ac.kr`)
- SMTP: Gmail (앱 비밀번호) 또는 학교 메일
- 형식: 종목별 PDF 첨부 + Summary 본문
- 인증: SMTP_USER / SMTP_PASS 환경변수

### 7.3 학회 드라이브 (Google Drive)
- 폴더: `/GIC_분석/위클리/[YYYY-WW]/[기업명]/`
- 자동 업로드: rclone 또는 Google Drive API
- 인증: OAuth2 (학회 운영진 계정)

## 8. 위험·검토 사항 (v10.1 이전 결정 필요)

1. **인증 보안**: Slack/Email/Drive 토큰을 어디에 저장할지 (학회 운영진 PC vs 클라우드 vault)
2. **MCP 비용**: tavily/exa 검색 API 무료 한도 vs 유료. 종목 N개 × 주 1회 = 비용 추정
3. **장애 대응**: cron 실패 시 알림 채널 (운영진 Slack DM?)
4. **부원 옵트아웃**: 발송 채널 구독 해제 메커니즘
5. **품질 게이트**: 자동 생성 yaml의 출처 누락·hallucination 점검 (블록 J) — 부원 확인 없이 발송하면 v7.0 재현 위험. 운영진 승인 단계 필수?

## 10. 결론

v10.0은 **위클리 자동 갱신의 모든 의존 자산을 준비**한다 (프롬프트·yaml 명세·Skill·HTML 템플릿). v10.1+에서는 **자동 정보 수집과 발송 채널만 추가**하면 된다.

CLAUDE.md §7.4 규정대로 *"자동화에 치중하면 베이스가 무너진다"*는 v7.0 교훈 박제 — 베이스(티어 1·2)가 단단하므로 자동 위클리(티어 5)는 **부원 검증을 거친 후 옵션으로** 도입.

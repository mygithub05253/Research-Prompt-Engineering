## 위클리 사용 워크플로

### 직전 분석이 없는 경우
1. 먼저 `GIC_기업리서치_v9.md`로 정식 9단계 풀 통과
2. Step 7 결과(투자의견·목표주가·6-Lens 점수) 저장
3. 이후부터 매주 위클리 사용

### 직전 분석이 있는 경우
1. 위 프롬프트 복사 + ■ 항목들 채우기
2. AI 챗봇에 붙여넣기
3. 답변 검토 → 4페이지 PPT 양식에 슬라이드별 복붙
4. 다음 주 위클리 시 이 답변을 "직전 위클리"로 사용

### 이번 주 새 정보가 거의 없을 때
"이번 주 정보 변동 없음. 직전 의견 유지" 1줄로 마무리. 4페이지 강제 채울 필요 없음.

## AI별 위클리 활용 팁 (본문 [점대괄호] 분기와 동일)

- GPTs Memory로 직전 위클리 자동 기억 (Plus)

### Gemini
- Search Grounding으로 출처 URL 자동 첨부 (가장 강력)
- 유튜브 분석 — 실적 발표·IR Day 영상 자동 트랜스크립트 (Advanced)
- Google Sheets 연동으로 누적 위클리 자동 기록 (Advanced)

## 부록 D — 위클리 4페이지 슬라이드 명세 (v10.0 신규)

> **목적**: 본 위클리 프롬프트 결과를 4슬라이드(또는 A4 4페이지) 양식에 1:1 매핑.
> **자동화**: `/gic-weekly [기업명]` Skill이 동일 변수 구조 사용. 트랙 3은 slides-grab으로 HTML→PDF/PNG 안정 출력 + anthropic-skills:pptx로 PPTX 부속 출력.
> **디자인**: `../design/GIC_design-guide_v10.md` §3·§5 토큰 적용. 위클리는 다크 표지 1p + 라이트 본문 3p 권장.

### D.1 슬라이드 1 — 표지 + 핵심 변동 (다크 톤 권장)

```yaml
slide_1_weekly_cover:
  메인_타이틀: "[≤25자, 형식: '[기업명] - [이번주 핵심 메시지]']"
                                                    # 예: "삼성전기 - 1Q OPM 서프라이즈"
  부제: "GIC Weekly | [YYYY-WW주차] | [YYYY.MM.DD]"
  학회기수: "GIC [N]기 [작성자]"

  투자의견_박스:                                       # 변동 시각화
    직전: "[BUY/HOLD/SELL]"
    이번주: "[BUY/HOLD/SELL]"
    화살표: "→"                                       # 변경 시 강조
    목표주가_직전: "[###,###원]"
    목표주가_이번주: "[###,###원]"
    상승여력: "[+##.#%]"
    주가_변동_W주W: "[+##.#%]"
    sixlens_점수_직전: "#.#"                          # 있으면
    sixlens_점수_이번주: "#.#"
```

## 1. 이 시스템은 무엇인가

가천대학교 GIC(Global Investment Club) 4기에서 만든 **AI 챗봇용 기업 리서치 프롬프트 시스템**입니다.

### 핵심 사양 (v11.0)
- **외부 의존 0**: API 키 / 웹앱 / MCP 서버 / Python 코드 모두 불필요
- **챗봇 복붙 100%**: Claude / ChatGPT / Gemini / Perplexity 모든 챗봇에서 동작
- **3종 산출물**: ① 정식 기업리서치 ② 위클리 (독립 분석) ③ 산업 Top Pick (사전 선정 종목 비교)
- **출력 형식 (v11.0 신규)**: HTML(A4 가로) + CSV + Excel + 프롬프트 메타 HTML

### 무엇을 만들 수 있나
- 증권사 PB급 정식 기업 리서치 리포트 (HTML 10~20p → 브라우저 인쇄로 PDF)
- 매주 신속 갱신 위클리 (HTML 5p, 핵심 압축)
- 사전 선정 종목 비교·추천 (HTML 8~13p)
- 5개년 재무 모델 csv·xlsx 파일
- 프롬프트 엔지니어링 메타 리포트 (별도 HTML)

### 무엇을 만들 수 없나
- 실시간 자동 매매 시스템
- DART/KRX API를 직접 호출하는 자동화 프로그램
- 학회 발표 양식 그대로의 PPT 파일 자동 생성 (HTML→PDF로 대체)

## 4. HTML→PDF 인쇄 가이드 (v11.0 핵심)

v11.0의 모든 산출물은 **HTML 파일**로 출력됩니다. 부원이 브라우저로 열고 인쇄하면 페이지가 정확히 분리된 PDF가 됩니다.

### 4.0 산출물별 페이지 모드 (중요)

| 산출물 | 페이지 모드 | 인쇄 옵션 |
|---|---|---|
| **위클리 투자리포트** | A4 **세로** (Portrait, 210×297mm) | 레이아웃: **세로** |
| **정식 기업리서치** | A4 **세로** (Portrait, 210×297mm) | 레이아웃: **세로** |
| **산업 Top Pick** | A4 **가로** (Landscape, 1280×720) | 레이아웃: **가로** |
| **프롬프트 메타** | A4 **세로** (기업리서치 부속) | 레이아웃: **세로** |

### 4.1 표준 워크플로 (5단계)

1. **AI 답변에서 HTML 코드 복사**
   - `<!DOCTYPE html>` 부터 `</html>` 까지 전체 선택

2. **`.html` 파일로 저장**
   - Windows: 메모장 또는 VS Code → 새 파일 → 붙여넣기 → "다른 이름으로 저장"
   - 파일명 끝에 `.html` (예: `삼성전기_GIC리서치_v11_20260514.html`)
   - 인코딩: **UTF-8** (한글 깨짐 방지)

3. **브라우저에서 열기**
   - 저장한 .html 파일을 더블클릭 → 기본 브라우저(Chrome/Edge/Safari)에서 열림
   - 페이지가 순차적으로 표시되는지 확인

4. **인쇄 → PDF 변환**
   - `Ctrl + P` (Mac: `Cmd + P`)
   - 대상: **PDF로 저장** (또는 Microsoft Print to PDF)
   - **레이아웃**: 산출물에 맞게 선택 (4.0 표 참조)
     · 위클리·기업리서치·메타 → **세로**
     · 산업 Top Pick → **가로**
   - **여백: 기본** (또는 없음)
   - **배경 그래픽**: ON (색상·뱃지 유지)
   - 저장 위치 지정 → 저장

5. **PDF 검증**
   - 모든 페이지가 1슬라이드 = 1페이지로 분리되었는지 확인
   - 표·박스가 페이지 경계에서 잘리지 않았는지 확인
   - 한글 폰트 깨짐 없는지 확인

### 4.2 페이지 잘림 현상 대응

v11.0의 CSS는 페이지 잘림을 방지하지만, 다음 경우는 추가 조치 필요:

| 증상 | 원인 | 해결 |
|---|---|---|
| 모든 페이지가 1페이지로 출력 | "맞춤"이 활성 | 인쇄 옵션에서 "실제 크기"·"100%" 선택 |
| 표가 페이지 끝에서 잘림 | 컨텐츠가 너무 많음 | AI에 "글자 수 ##% 줄여 재출력" 요청 |
| 한글 폰트가 □□로 표시 | UTF-8 미적용 | .html 저장 시 UTF-8 인코딩 명시 |
| 색상 박스가 회색으로 출력 | 배경 그래픽 OFF | 인쇄 옵션 "배경 그래픽" ON |
| 페이지 사이 빈 페이지 추가 | 마지막 페이지 page-break | AI에 "마지막 .page에 last-child CSS 적용 확인" |

### 4.3 권장 브라우저

| 브라우저 | A4 가로 인쇄 | CSS @page 지원 | 한글 폰트 |
|---|---|---|---|
| **Chrome** ⭐ | ✓ 완벽 | ✓ | ✓ |
| **Edge** ⭐ | ✓ 완벽 | ✓ | ✓ |
| **Safari** | ✓ | ✓ | ✓ |
| **Firefox** | △ (여백 조정 필요) | △ | ✓ |
| **Whale (Naver)** | ✓ | ✓ | ✓ |

## 5. 4개 파일의 역할

### 📘 GIC_기업리서치_v11.md (메인)
- **줄 수**: 약 800줄
- **구조**: 10단계 (Step 0~9 + Step 5.5) + 부록 A(Quick Track) + 부록 B(블록 라이브러리)
- **사용 빈도**: 종목당 1회 풀 분석 (이후 위클리로 갱신 옵션)
- **산출물**: HTML 10~20p + CSV 7개 + (옵션) Excel + 메타 HTML 2~3p
- **핵심**: 정식 시스템의 모든 것이 이 한 파일에

### 📗 GIC_위클리투자리포트_v11.md (재정의)
- **줄 수**: 약 300줄
- **구조**: 단일 압축 프롬프트 + 5페이지 양식
- **사용 빈도**: 종목당 매주 1회 (또는 의사결정 필요 시)
- **사전 조건**: **없음 — 정식 분석 없이도 단독 사용 가능 (v11.0 변경)**
- **산출물**: HTML 5p + CSV 3개
- **핵심**: 실전 압축 — 매수/매도/홀드 + Point 3 + 재무 요약 + 판단 근거 + 참고문헌

### 📙 GIC_산업TopPick_v11.md (방향성 전환)
- **줄 수**: 약 400줄
- **구조**: Phase 1~8 (산업 진단 → 선정 종목 비교 → 심층 카드 → 6-Lens 비교 → Risk-Reward → Red Team → 모니터링 → 메타)
- **사용 빈도**: 산업 변곡점마다 / 조 사전 토론 후
- **사전 조건**: **조에서 사전 선정한 2~5개 종목 (v11.0 핵심 변경)**
- **산출물**: HTML 8~13p + CSV 3개
- **핵심**: AI의 종목 발굴이 아닌 — 조의 선정 종목 객관 비교 + 추천 강화

### 📕 GIC_사용설명서_v11.md (본 문서)
- **줄 수**: 약 600줄
- **구조**: 진입 가이드 + HTML→PDF + 데이터 수집 + FAQ + 트러블슈팅 + 변경이력
- **사용 빈도**: 처음 1회 정독 + 트러블슈팅 시 검색

### ChatGPT (chatgpt.com)
- **수동 활성**: 메시지 입력란의 **도구** 또는 + 아이콘 클릭 → "웹 검색" 토글 ON
- GPT-4o·5 등 최신 모델에서 사용 가능
- 무료 사용자도 일일 한도 내 가능

### Gemini (gemini.google.com)
- **자동 ON (Search Grounding)**: 기본 활성
- 답변 끝에 출처 URL이 자동 첨부됨

### Perplexity (perplexity.ai)
- **기본 검색 엔진**: 모든 답변이 웹 검색 기반
- Pro 모드에서 GPT-4·Claude·Sonar 선택 가능

### 웹검색이 안 될 때
1. 챗봇 재시작
2. 다른 챗봇으로 시도
3. 프롬프트에 "웹검색 활성 확인 후 답변" 명시
4. 답변 첫 줄에 "웹검색 비활성" 경고가 있으면 → 활성화 후 재시도

연도,매출,영업이익,순이익,OPM(%),ROE(%)

```
```

**저장 방법**:
1. ` ```csv` 다음 줄부터 마지막 ` ``` ` 직전까지 텍스트 선택·복사
2. 메모장 또는 VS Code 열기 → 붙여넣기
3. "다른 이름으로 저장" → 파일명 끝에 `.csv`
4. 인코딩: **UTF-8** (한글 깨짐 방지)
5. 더블클릭 → Excel에서 바로 열림

### 8.2 Excel(.xlsx) 자동 생성 — Code Interpreter

다음 챗봇 환경에서 자동 .xlsx 다운로드 지원:
- **ChatGPT Plus** (Code Interpreter) ⭐
- **Claude Pro** (Computer Use 또는 Artifacts)
- **Gemini Advanced** (Code Execution)

프롬프트 안에 "Code Interpreter 사용 가능 시 .xlsx 파일 생성"이 명시되어 있어 자동 실행됨.

다운로드 링크가 답변에 표시되면 → 클릭 → Excel 열기.

### 8.3 csv vs Excel 사용 가이드

| 상황 | 추천 |
|---|---|
| 데이터만 빠르게 보기 | csv (어떤 환경이든 호환) |
| 수식·차트·시각화 | Excel (Code Interpreter 활성 챗봇) |

| 학회 발표 자료 | Excel (서식·차트 포함) |

## 9. 자주 묻는 질문 (FAQ)

### Q1. AI 챗봇 어떤 걸 써야 하나?
A. 첫 분석은 **Claude 무료** 또는 **ChatGPT** (웹검색 활성 후). 익숙해지면 본인 선호로. 웹검색 강도는 Gemini > Perplexity > ChatGPT ≈ Claude 순.

### Q2. 무료 AI에서도 정식 10단계 가능?
A. 가능. Claude 무료는 일일 메시지 한도, ChatGPT 무료는 GPT-4o-mini 제한 있음. 단 Code Interpreter(Excel 자동 생성)는 ChatGPT Plus/Claude Pro 필요.

### Q3. 사업보고서 PDF 100페이지 넘는데 AI가 다 읽나?
A. Claude(200K) / Gemini(2M) — 100p+ OK. ChatGPT — 50~80p 안전선, III장 재무·II장 사업만 잘라 첨부.

### Q4. HTML이 PDF로 잘 변환 안 돼요
A. 다음 순서로 확인:
1. 브라우저는 Chrome/Edge 권장
2. 인쇄 옵션 → "레이아웃: 가로" 확인
3. "배경 그래픽" ON
4. "여백: 기본"
5. 그래도 안 되면 본 문서 "4.2 페이지 잘림 현상 대응" 참조

### Q5. csv가 Excel에서 한글이 깨져요
A. csv 저장 시 인코딩을 **UTF-8**로 명시. Excel에서 한글 깨지면:
1. Excel → 데이터 → 텍스트/CSV 가져오기
2. 원본 파일 → UTF-8 선택
3. 미리보기 확인 후 가져오기

### Q6. 위클리에 정식 분석이 없어도 되나요? (v11.0 변경 확인)
A. **예, 됩니다**. v9.0까지는 정식 분석 결과를 갱신하는 도구였지만, v11.0의 위클리는 **독립 압축 분석** 도구로 재정의되었습니다. 종목명만 입력하면 5p HTML이 나옵니다.

### Q7. 산업 Top Pick에서 AI가 종목을 발굴해주나요? (v11.0 변경 확인)
A. **아니요, v11.0에서는 조에서 사전 선정한 종목 2~5개를 입력**해야 합니다. AI는 비교·심층 분석·추천 강화에 집중합니다. v9.0 방식(AI 스크리닝)은 폐기되었습니다.

### Q8. 6-Lens 가중 점수가 상승여력 룰과 충돌하면?
A. Step 7-3에서 사유 명시. 일반적으로 정성 요소(Moat·Promoter Behavior)가 강하면 6-Lens 우선.

### Q9. AI별 답변이 너무 다른데 일관성 어떻게?
A. v11.0은 본문에 [Claude:] [ChatGPT:] [Gemini:] 점대괄호 분기로 차이를 흡수. 골격은 같음.

### Q10. Quick Track과 정식 10단계의 차이?
A. Quick Track(부록 A) — PDF 1개로 10분, 사전 스크리닝용. 정식 10단계 — 1시간 10분, 학회 발표용.

### Q11. 학회 PPT 양식과 HTML이 안 맞아요
A. v11.0의 7페이지 표준 양식은 가이드. 학회 양식이 5페이지면 페이지 합치고 조정. 핵심은 **글자 수 룰 + 6-Lens 점수 + Evidence Card** 유지.

### Q12. 프롬프트 메타 리포트(Step 9)는 꼭 만들어야 하나요?
A. 학회 발표 시 "이 리포트는 어떤 AI 프롬프트로 만들어졌는가" 백업 자료로 권장. 필수는 아니지만, 발표자 신뢰도 ↑.

### 문제 1: 코드블록이 잘려서 나옴
**원인**: 마크다운 뷰어가 4중 백틱 미지원 / AI 응답 토큰 한도

### 문제 2: AI가 출처 없는 수치를 자꾸 만들어냄
**원인**: 블록 J(Anti-Hallucination Protocol)을 적용하지 않음 + 웹검색 비활성
**해결**:
1. Step 0의 [블록 J 적용] 박스를 명시적으로 강조
2. 웹검색 활성 재확인 (W원칙)
3. 모든 핵심 수치에 (출처: OOO, YYYY.MM, URL) 의무화

### 문제 3: HTML이 한 페이지로 출력됨
**원인**: 인쇄 옵션 "맞춤" 활성 / CSS 미적용

1. 인쇄 → "실제 크기" 또는 "100%" 선택
2. "여백: 기본" 또는 "없음"
3. 그래도 안 되면 AI에 "page-break-after: always가 모든 .page에 적용됐는지 확인" 재요청

### 문제 4: HTML 내용이 페이지 경계에서 잘림
**원인**: 컨텐츠 양이 한 페이지(297×210mm)를 초과

1. AI에 "글자 수 NN%로 줄여 재출력" 요청
2. 또는 표·박스를 다음 페이지로 분리
3. 글자 크기를 9pt → 8.5pt로 축소 요청

### 문제 5: csv 파일이 Excel에서 한 컬럼에 다 들어감
**원인**: 구분자 인식 실패 (특히 한글 Excel)
**해결**:
1. Excel → 데이터 → 텍스트 나누기 → 쉼표 구분
2. 또는 csv 저장 시 UTF-8 BOM 인코딩
3. 또는 Google Sheets에서 열기 (자동 인식)

**원인**: 두 룰이 다른 차원을 측정

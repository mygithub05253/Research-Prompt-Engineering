> **작성일**: 2026-05-14

## 1. 변경 배경 — 사용자 피드백 8건

GIC 4기 운영팀이 v9.0 사용 + v10.0 시범 운용 후 수집한 피드백 8건:

| # | 피드백 | v11.0 대응 |
|---|---|---|
| 1 | 재무 분석 시 csv/Excel로 실제 통계 파일 확인하고 싶음 | **X원칙 신설** — csv 코드블록 + Code Interpreter Excel 동시 출력 |
| 2 | 각 단계마다 분석 결과물을 HTML로 만들면 좋겠음 | **P원칙 신설** — Step 8 HTML 양식 + 단계별 결과 HTML 임베드 가능 |
| 3 | PDF/PPTX를 한번에 만들지 말고 HTML→인쇄→PDF 가로로. 단 잘림 현상 발생 | **A4 가로 CSS @page + page-break-after + break-inside: avoid** 표준화 |
| 4 | 마지막에 프롬프트 엔지니어링 설명 슬라이드 추가 | **Step 9 + 별도 메타 HTML** (M원칙) 신설 |
| 5 | 위클리는 기업리서치 요약이 아니라 별개 핵심 분석이어야 | **위클리 독립 재정의** — 정식 분석 없이도 5p 핵심 압축 가능 |
| 6 | 산업 Top Pick은 조에서 사전 선정한 종목으로 진행 | **Top Pick 방향성 전환** — AI 발굴 X / 조 사전 선정 2~5개 비교·분석 |
| 7 | 웹크롤링·최신 주가 반영 필수 | **W원칙 신설** — 챗봇 내장 웹검색 의무화 + URL 첨부 |
| 8 | HTML→PDF 변환 시 1페이지로 압축되거나 잘리는 현상 | **CSS 페이지 분할 표준화** + 사용설명서 4.2 트러블슈팅 |

## 2. v10.0 폐기 사유 (재발 방지 기록)

v10.0은 다음과 같은 방향성으로 시도됨:
- Python 스크립트(`render_research.py`) 기반 PPT 자동 생성
- `.claude/skills/gic-research/` 스킬 시스템 도입
- 디자인 양식 분석 + 자동 렌더링

### 폐기 사유

1. **부원 진입 장벽 ↑**
   - Python 환경 설치 필요 (Anaconda or python.org)
   - 의존 라이브러리 설치 (python-pptx, pandas 등)
   - 비전공자 부원의 30분 소요 → 30초 시작 원칙 위배

2. **학회 PPT 양식과 호환성 문제**
   - 자동 생성 PPT가 학회 표준 양식과 미세하게 불일치
   - 결과적으로 부원의 수동 조정 부담 ↑

3. **AI 모델 종속성**
   - 특정 챗봇 환경에 종속되는 흐름 발생
   - "AI 모델·웹앱·MCP가 사라져도 잘 쓴 .md 프롬프트는 영원하다" 철학 위배

4. **v7.0 실패의 재발**
   - v7.0이 `gemini-2.0-flash` 모델 의존으로 폐기되었던 사례
   - v10.0의 Python 의존은 같은 종류의 종속

### 교훈
> **자동화에 치중하면 베이스가 무너진다** — v7.0 폐기 시 도출된 핵심 교훈을 v11.0이 재확인.

## 3. v9.0 → v11.0 마이그레이션 가이드

### 3.1 v9.0 사용자가 v11.0으로 옮길 때

| 작업 | v9.0 방식 | v11.0 방식 |
|---|---|---|
| **종목 분석 시작** | `GIC_기업리서치_v9.md` Step 0 복사 | `GIC_기업리서치_v11.md` Step 0 복사 (웹검색 활성 확인 추가) |
| **재무 데이터** | 마크다운 표만 출력 | 마크다운 표 + csv 코드블록 동시 출력 |
| **출력 형태** | 텍스트 복붙 가이드 → 부원이 PPT 수동 작업 | HTML 코드 → .html 저장 → 브라우저 인쇄 → PDF |
| **위클리 사용** | 정식 분석 결과 갱신 | 정식 분석 없이도 단독 5p 분석 가능 |
| **산업 Top Pick** | 후보 5~10개 AI 스크리닝 | 조 사전 선정 2~5개 비교·분석 |
| **웹검색** | 권장 (모델별 자동) | 의무 (W원칙) |
| **메타 리포트** | 없음 | Step 9 + 별도 HTML |

### 3.2 단계별 마이그레이션 (10분)

1. **v11.0 폴더 확보** — `version11.0/` 폴더 다운로드
2. **사용설명서 정독** — `GIC_사용설명서_v11.md` "v9.0 → v11.0 핵심 차이" 표
3. **첫 분석 시도** — 익숙한 종목으로 `GIC_기업리서치_v11.md` 1회 실행
4. **HTML 변환 확인** — Step 8 HTML 코드 → 브라우저 → PDF 가로 인쇄
5. **csv 확인** — 답변의 csv 코드블록을 .csv로 저장 → Excel에서 열기
6. **메타 리포트 생성** — Step 9 별도 HTML로 출력

## 4. v11.0 4개 신규 횡단 원칙 (W·X·P·M)

### W — Web Search (웹검색 의무화)
- **무엇**: 모든 시장 가격·시총·최신 뉴스는 챗봇 내장 웹검색 결과 인용 + URL 첨부
- **왜**: v9.0의 "권장"으로는 일부 부원이 비활성 상태로 사용 → 추측 답변 발생
- **어디**: Step 0(초기), 1(피어 시총), 3(산업), 4(뉴스), 6(피어 멀티플), 위클리 전체, Top Pick 전체

### X — eXport (CSV/Excel 동시 출력)
- **무엇**: 모든 핵심 표를 csv 코드블록으로 동시 출력 + Code Interpreter 사용 시 .xlsx 자동
- **왜**: 부원이 실제 데이터 파일로 검증·재사용 가능해야 함
- **어디**: Step 1(피어), 2(재무 3종), 3(시장 규모), 4(사업부 매출), 6(밸류 4종), 7(6-Lens)

### P — Page-safe HTML (페이지 분할 안전)
- **무엇**: 양식별 페이지 모드 + CSS `page-break-after: always` + `break-inside: avoid` 표준화
- **양식별 모드** (학회 양식 첨부 베이스):
  - **위클리·기업리서치·메타** → **A4 세로** (Portrait, 210×297mm 또는 1000px 너비)
  - **산업 Top Pick** → **A4 가로** (Landscape, 16:9 1280×720 슬라이드)
- **왜**: v9.0의 텍스트 복붙 방식 → PPT 수동 작업 → 페이지 잘림 발생
- **어디**: Step 8 (정식 9p), 위클리 (5p), Top Pick (10p), 메타 (3p)

### M — Meta Prompt (프롬프트 설명 리포트)
- **무엇**: 사용한 프롬프트 기법(블록 A~J, 6-Lens, AD-FCoT)을 별도 HTML로 설명
- **왜**: 학회 발표 시 "이 리포트는 어떤 AI 프롬프트로 만들어졌는가" 백업 자료
- **어디**: Step 9 (기업리서치 끝에 추가), Top Pick Phase 8

## 6. v11.0 8대 원칙 (요약 재게시)

1. **HTML 우선** — 모든 산출물은 HTML(A4 가로) 우선 → 부원이 브라우저 인쇄로 PDF
2. **데이터 파일 동시 출력** — csv 코드블록 + Excel 자동 생성
3. **웹검색 의무화** — 챗봇 내장 웹검색으로 최신 데이터 + URL
4. **페이지 분할 안전 CSS** — A4 가로 + page-break + break-inside avoid
5. **위클리 독립** — 정식 분석 없이도 단독 핵심 압축
6. **Top Pick 사전 선정** — 조에서 종목 입력 → 비교·분석
7. **프롬프트 메타 공개** — 사용 기법을 본문 끝 + 별도 HTML로 설명
8. **외부 의존 0** — API 키·웹앱·MCP 모두 불필요. 챗봇 복붙만으로 동작

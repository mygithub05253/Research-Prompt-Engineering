## v7.0 → v8.0 상세 변경 사유

### 1. 웹앱(index.html) 폐기 결정

**문제**: v7.0의 index.html이 `gemini-2.0-flash`를 직접 호출.
- 해당 모델은 2026-03-03자로 deprecated → free tier limit이 0이 됨
- 사용자가 받은 429 오류 = "한도 초과"가 아니라 "모델이 무료 티어에서 빠짐"
- importmap의 `@anthropic-ai/genai` 표기 오류
- 사용자가 API 키 발급해야만 동작하는 구조 → 부원 진입 장벽

**해결**: 웹앱 자체를 폐기.
- v8.0은 마크다운 파일만 배포
- AI 챗봇(Claude / ChatGPT / Gemini Chat)에 복붙으로 100% 동작
- API 키·MCP 서버·Python 코드 0 의존

### 2. v5.0의 9단계 골격 회귀

**문제**: v7.0이 7단계로 압축하면서 단계별 가이드가 너무 짧아짐.
- 비전공자가 챗봇에 복붙해도 한 번에 끝 → 단계별 검토 어려움
- AD-FCoT·Evidence Card 같은 좋은 자산은 있는데 흡수가 추상적

**해결**: v5.0의 9단계(Step 0~8 + 5.5) 골격 그대로 복원.
- 각 단계가 ``` 코드 블록 ``` 안에 완전한 프롬프트로 들어감
- 사용자는 그 블록을 그대로 챗봇에 붙여넣기만 하면 됨

### 3. v7.0 자산을 횡단 블록으로 흡수

**v5.0 블록 (유지)**: A·B·C·D·E·F (6개)
**v8.0 신규 블록 (3개 추가)**:
- **G — Evidence Card** (v7의 Evidence Card 횡단화)
- **H — AD-FCoT** (v7의 AD-FCoT 횡단화 + 차이점 명시 강제)
- **I — 5개년 Forward 모델링** (v7의 Blended CAGR 흡수 + 3-Statement 연결 표준화)

→ 단계 골격은 v5.0, 깊이는 v7.0 흡수. 두 마리 토끼 동시에.

### 4. Step 6 5개년 모델링 핵심 강화

**v5.0 Step 6**: PER × Peer 평균이 주력, DCF는 보조
**v6.0 Step 6**: 3-Statement 연결 + Excel 시트 7개
**v7.0 Step 4**: Evidence Card + Blended CAGR + 시나리오 매트릭스
**v8.0 Step 6**: 위 셋 모두 흡수

- 가정표 (Bear/Base/Bull)
- Blended CAGR 공식 (50% 1Y + 30% 3Y + 20% 5Y)
- 3-Statement 연결 + Balance Check
- 4종 밸류에이션 (PER·PBR·EV/EBITDA·DCF)
- 시나리오 매트릭스 (FY1~FY5 × 3 시나리오)
- 민감도 분석 (EPS × PER 매트릭스)
- 투자의견 자동 판정
- 모니터링 체크리스트

### 5. Quick Track 신설 (yj.builder 벤치마크)

**새 파일**: `Quick_Track_5Y_Model.md`

벤치마크: yj.builder Threads 글 ("5천억짜리 기업 재무 모델링, 이제 10분 컷")
- 사업보고서 PDF 1개 + 한 명령
- 7개 Phase 12분 가정
- 정식 9단계 코스 진입 전 사전 스크리닝용

### 6. 한국 환경 데이터 가이드 분리

**새 파일**: `데이터수집가이드.md`

DART / KRX / 네이버금융 / FnGuide / 인베스팅닷컴 / 증권사 리서치 / 산업 통계 / 거시 / 뉴스 — 모두 무료 출처. 코드 없음.

## v5.0 → v8.0 호환성

v5.0의 9단계 + 블록 A~F를 그대로 사용하던 부원은 v8.0에서 **재학습 불필요**.
신규 블록 G·H·I는 자동 발동되며, 어색하면 Step 0에서 "블록 G·H·I 비활성"으로 끄면 v5.0과 동일하게 동작.

## 블록 A — 용어 번역기

### 정의
전문용어가 본문에 처음 등장하는 위치에 **(10자 내외 쉬운 풀이)** 를 괄호로 부착.

### 작동 규칙
1. **첫 등장 위치에만** 풀이를 부착 (반복 시 생략)
2. 풀이는 **10자 내외**로 압축
3. 영어 약어는 한글 풀이 우선, 영어 원어는 후순위
4. 부원이 "블록 A 비활성"이라고 명시하면 즉시 끄기

### 예시
| 용어 | 풀이 |
|---|---|
| ROE | 자기자본 대비 이익률 |
| ROIC | 투자자본 대비 이익률 |
| CAPEX | 설비투자 지출액 |
| OPM | 영업이익률 |
| EBITDA | 감가 전 영업이익 |
| FCF | 잉여현금흐름 |
| WACC | 가중평균자본비용 |
| EV/EBITDA | 기업가치 ÷ 감가전이익 |
| PER | 주가 ÷ 주당순이익 |
| PBR | 주가 ÷ 주당순자산 |
| CCC | 현금전환주기 |
| Moat | 경쟁우위 해자 |
| ASP | 평균판매단가 |
| YoY | 전년동기 대비 |
| CAGR | 연평균 성장률 |
| NTM | 향후 12개월 |
| SOTP | 사업부별 합산 |

### 출력 형식
> "**ROE(자기자본 대비 이익률)** 는 12.3%로 동종 평균 8.5% 대비 우위에 있다."

## 블록 B — Sanity Check

### 정의
출력 직전 자체 점검으로 **1차원 인과 비약**을 적발하고 매개변수를 보강.

### 점검 항목
□ "매출↑→주가↑" 같은 1차원 인과 없음
□ "시장 성장 = 자동 성장" 가정 없음
□ "신제품 출시 → 실적 개선" 비약 없음
□ "멀티플 리레이팅 = 자동" 가정 없음
□ 점유율·마진·멀티플 매개변수 명시
□ 가정 변경 시 어디서 어떻게 영향받는지 분해

### 보강 형식
취약 문장 발견 시 다음 패턴으로 재작성:
> "[가정] → [매개변수 분해] → [수치 영향]"

예) ❌ "AI 시장 성장으로 매출 증가"
   ✅ "AI 서버용 MLCC 점유율 35% 가정 시, 글로벌 AI 시장 CAGR +25% 중 우리 매출 기여 = 35% × 25% × 단가 인상분 +10% = +35.4% YoY"

## 블록 C — 핵심 비유

### 정의
산업·기업의 핵심 정체성을 **30자 이내 1줄 비유**로 본문 최상단에 배치.

### 형식
"이 회사는 [산업]의 [대중적 서비스] 같은 존재 — [핵심 차별점]"

### 예시
- "이 회사는 반도체 업계의 스타벅스 — 매장(Fab)을 직접 운영"
- "이 회사는 조선의 테슬라 — 친환경 LNG선 글로벌 1위"
- "이 회사는 K-바이오의 삼성전자 — CDMO 풀라인업 보유"
- "이 회사는 제약의 코스트코 — 박리다매 제네릭 1위"
- "이 회사는 부품의 인텔 — 스마트폰 두뇌 AP 핵심 공급"

### 효과
- 비전공자가 5초 내에 "이 회사 뭐하는 곳인지" 이해
- 발표 슬라이드 P.3 좌측 하단 박스에 그대로 사용 가능

## 블록 E — 검증 태그

### 정의

### 발동 조건

## 블록 G — Evidence Card (v8.0 신규)

### 정의
모든 핵심 가정에 **근거 3건 + 출처 + 신뢰도**를 카드 형식으로 부착.
v7.0의 핵심 자산을 횡단 블록으로 재배치.

### 카드 형식

```
┌─────────────────────────────────────────────┐
│ 가정: [한 문장으로 가정 명시]
│
│ 근거 ①: [구체 사실/수치] (출처: OOO, YYYY.MM)
│ 근거 ②: [구체 사실/수치] (출처: OOO, YYYY.MM)
│ 근거 ③: [구체 사실/수치] (출처: OOO, YYYY.MM)
│
│ 신뢰도: [상/중/하]
│   - 상: 3건 일치 + 1차 출처
│   - 중: 2건 일치 + 추정 1건
│   - 하: 1건 또는 모두 추정
└─────────────────────────────────────────────┘
```

### 발동 위치
- Step 5: 모든 투자포인트
- Step 6: 모든 핵심 가정 (매출 성장률, OPM, CAPEX, WACC, g)
- Step 4: Moat 요소 판정

### 예시
```
가정: 삼성전기 MLCC 사업부 FY1 매출 +18% YoY
근거 ①: AI 서버용 고용량 MLCC 채택 가속 (출처: 삼성전기 IR, 2026.04)
근거 ②: 글로벌 MLCC 시장 CAGR +12% (출처: TrendForce, 2026.03)
근거 ③: 점유율 22% → 24% 확장 가이던스 (출처: 컨퍼런스콜, 2026.04)
신뢰도: 상 (3건 일치, 1차 출처 2건)
```

### 신뢰도 판정 알고리즘
1. 근거 3건이 모두 같은 방향을 가리키는가? (Yes/No)
2. 근거 중 1차 출처(공시·기업 IR·정부 통계) 비중이 절반 이상인가? (Yes/No)
3. 근거의 시점이 최근 6개월 이내인가? (Yes/No)

3개 모두 Yes → **상**
2개 Yes → **중**
1개 이하 Yes → **하**

### 좋은 예시

>  ※ 차이점: ① SK하이닉스는 메모리 단일 사업, 삼성전기는 MLCC + FC-BGA 다각화
>            ② 당시 코로나 비대면 수요라는 외생변수 존재"

### 나쁜 예시 (단순 유추 — 금지)

### 자체 점검

□ 차이점 1~2개가 명시되어 있는가?

## 블록 I — 5개년 Forward 모델링 (v8.0 신규)

### 정의
v7.0의 Blended CAGR을 흡수하고 3-Statement 연결까지 표준화한 횡단 블록.

### 발동 위치
Step 6 밸류에이션의 핵심 엔진.
선택적으로 Step 5 투자포인트의 정량 근거에서도 호출 가능.

### Blended CAGR 공식
```
사업부별 매출 성장률 = 50% × 1Y Growth
                    + 30% × 3Y CAGR
                    + 20% × 5Y CAGR
```

근거:
- **50% 1Y**: 단기 추세가 가장 최근 정보
- **30% 3Y**: 중기 트렌드 균형
- **20% 5Y**: 장기 사이클 평균 회귀
- 가중치는 분석가 재량 조정 가능 (단, 합 = 100%)

### 3-Statement 연결 체크리스트

**손익계산서 (IS)**
□ 사업부별 매출 합 = 전체 매출
□ 매출원가율·판관비율 가정 명시
□ 영업외손익·세금 가정 명시
□ 지배순이익 = 당기순이익 - 비지배지분
□ EPS = 지배순이익 ÷ 발행주식수

**재무상태표 (BS)**
□ 운전자본 = 매출채권 + 재고 - 매입채무
□ 유형자산 = 기초 + CAPEX - 감가상각
□ 이익잉여금 = 기초 + 당기순이익 - 배당
□ **자산 = 부채 + 자본** (Balance Check)

**현금흐름표 (CF)**
□ 영업CF = 순이익 + D&A ± 운전자본 변동
□ 투자CF = -CAPEX ± 기타투자
□ 재무CF = 차입금 변동 - 배당
□ **FCF = 영업CF - CAPEX**
□ 기말 현금 = 기초 + 영업CF + 투자CF + 재무CF
